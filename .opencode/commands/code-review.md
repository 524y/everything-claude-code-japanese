# コードレビュー

未コミットの変更に対して包括的なセキュリティ/品質レビューを行う:

1. 変更されたファイルを取得: git diff --name-only HEAD

2. 変更された各ファイルについて次を確認:

**セキュリティ問題（CRITICAL）:**
- ハードコードされた認証情報、API キー、トークン
- SQL injection の脆弱性
- XSS の脆弱性
- 入力検証の不足
- 安全でない依存関係
- パストラバーサルのリスク

**コード品質（HIGH）:**
- 50 行超の関数
- 800 行超のファイル
- 4 段超のネスト
- エラーハンドリング不足
- console.log の記述
- TODO/FIXME コメント
- 公開 API の JSDoc 不足

**ベストプラクティス（MEDIUM）:**
- 破壊的変更のパターン（immutable を使う）
- コード / コメント内の絵文字使用
- 新規コードのテスト不足
- アクセシビリティ問題（a11y）

3. 次を含むレポートを作成:
   - 重要度: CRITICAL, HIGH, MEDIUM, LOW
   - ファイル位置と行番号
   - 問題の説明
   - 修正案

4. CRITICAL または HIGH が見つかったら commit をブロック

セキュリティ脆弱性があるコードは決して承認しないこと!
