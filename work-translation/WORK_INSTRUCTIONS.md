# 翻訳作業指示書

## 目的
この指示書は、`en/` にある everything-claude-code の指示文書を `ja/` に同一ツリー構造で翻訳するための作業手順とルールを定義する。
コンテキストの爆発を抑え、文書の統一感を維持しながら複数セッションで安全に進める。

## 翻訳対象と除外
- 対象
  - `**/*.md` と `**/*.tmp` は全文を翻訳する
  - `**/*.js` / `**/*.sh` / `**/*.py` はコメントのみ翻訳する
- `**/*.json` は `description` / `comment` / `help` / `title` / `_comments` の値のみ翻訳する
- 除外
  - `LICENSE` は翻訳しない
  - 画像 / 動画 / バイナリ は対象外

## 文体と表記ルール
- 文体は「である」調
- 日本語と英単語の間に半角スペースを入れる
- 日本語の単語同士にスペースを入れない
- 行末の不要な空白は削除する
- 文末に改行を入れる
- 半角数値と全角の単位や全角文字の間は半角スペースを入れる（例: 200-400 行、800 行）
- `A / B / C` のようにスラッシュ列挙の前後に半角スペースを入れる
- `80% + カバレッジ` のように `+` の前後に半角スペースを入れる

## 形式維持ルール
- Markdown の見出し階層 / 箇条書き / 番号 / 引用 / 表 / 区切り線を維持する
- コードブロック内は原則 翻訳しない。ただし *.md のコードブロック内に含まれるコメントは翻訳する
- *.md のコードブロック内に別の Markdown が含まれる場合は、Markdown のルールに従って翻訳する
- 使用例 / 出力例 / テンプレのコードブロックで自然言語の指示や説明が中心の場合は翻訳する（識別子 / 固有名詞 / コマンド / パス / URL / ファイル名は原文維持）
- コードブロック内の見出し / ラベル / 説明文など自然言語の本文は翻訳する（言語が `text` / `markdown` / 未指定のコードブロックも対象）
- Markdown 内の **...** で示されるラベルは翻訳する（識別子 / 固有名詞は原文維持）
- コード内コメントの GOOD: / BAD: / CORRECT: / WRONG: / FIX: / FIX ERROR: などのキーワードは原文維持する
- DB とプログラミング言語のデータ型名（string, number, boolean, int, bigint, numeric, timestamptz など）は原語のままにする
- SQL はコードブロックと同じルールを適用する（GOOD: などのキーワードは原文維持）
- レビュー出力形式などのテンプレ内は本文を日本語化し、ラベル/キーワード（CRITICAL/HIGH/MEDIUM など）は原文維持する
- インラインコード / パス / コマンド / URL / ファイル名は原文維持
- 変数名 / クラス名 / 関数名は原文維持
- ディレクトリ名 / コマンド名など識別子として使われる英語は原文維持

## 用語統一
- 翻訳開始前に用語表を作成し、全セッションで参照する
- 新しい用語が出たら用語表に追記する
- 迷う訳語は採用前に一度メモし、後から全体で統一する

### 用語表例
- command: コマンド
- rule: ルール
- hook: フック
- agent: エージェント
- skill: スキル
- context: コンテキスト
- review: レビュー
- test: テスト

## 複数セッションでの進め方
- 1 セッションで扱う範囲を限定し、対象ファイルの一覧を必ず明示する
- 未翻訳一覧は `UNTRANSLATED_FILES.md` で管理し、完了時に削除する
- 進捗管理ファイルを `TRANSLATION_STATUS.md` として作成し、各セッション終了時に更新する
- `TRANSLATION_STATUS.md` に以下を記録する
  - セッション番号
  - 対象ファイル一覧（`UNTRANSLATED_FILES.md` への参照でもよい）
  - 完了 / 未完了
  - 用語表の更新有無
  - 翻訳時の注意点
- 既存の翻訳済みファイルは再翻訳しない
- 既存ファイルに差分が生じた場合のみ追従する

## セッション終了時の Codex 側の指示
- そのセッションで処理したファイル一覧を明示する
- 未完了の項目があれば明記する
- `TRANSLATION_STATUS.md` を更新したかどうかを報告する
- `UNTRANSLATED_FILES.md` から削除したかどうかを報告する
- 追加 / 変更した用語があれば `TERM_GLOSSARY.md` の更新内容を記載する
- 次セッションの開始点を 1 行で提示する

## 作業手順
1. 対象ファイルを一覧化する
2. 初期ルール作成として、`en/everything-claude-code/README.md` と各ディレクトリから 1 ファイルずつ参照する
3. 用語表を確認し、足りない用語を追加する
4. 1 ファイルずつ翻訳する
   - Markdown は段落単位で翻訳
   - コメント翻訳は意味を変えずに短くする
5. `ja/` に同一パスで保存する
6. 差分確認を行い、形式崩れがないか確認する
7. 翻訳完了ファイルは `UNTRANSLATED_FILES.md` から削除する
8. `TRANSLATION_STATUS.md` に記録する

## 翻訳ルール作成手順
- まず `TERM_GLOSSARY.md` を作成し、用語表を記載する
- 次に `en/everything-claude-code/README.md` と各ディレクトリの代表ファイルで試験翻訳し、用語表を調整する
- 用語表が安定したら全体へ適用する

## 品質チェック
- Markdown の構造が保持されているか確認する
- コードブロックの中が無断で翻訳されていないか確認する（*.md のコメント翻訳は除く）
- URL / パス / コマンドが変わっていないか確認する
- 用語が用語表と一致しているか確認する
