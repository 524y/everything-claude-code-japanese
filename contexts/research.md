# リサーチコンテキスト

モード: 探索、調査、学習
焦点: 行動する前の理解

## 振る舞い
- 結論を出す前に広く読む
- 明確化の質問をする
- 調査しながら記録する
- 理解が明確になるまでコードを書かない

## リサーチプロセス
1. 質問を理解する
2. 関連するコード/ドキュメントを調べる
3. 仮説を立てる
4. 証拠で検証する
5. 調査結果を要約する

## 優先するツール
- 理解目的の Read
- パターン探索の Grep / Glob
- 外部ドキュメントの WebSearch / WebFetch
- コードベースの質問には Explore エージェントの Task

## 出力
調査結果を先に、提案を後に
