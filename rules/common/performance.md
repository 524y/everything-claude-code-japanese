# パフォーマンス最適化

## モデル選定戦略

**Haiku 4.5**: 頻繁な軽量タスク向け

**Sonnet 4.5**: 主開発とオーケストレーション向け

**Opus 4.5**: 深い推論が必要な難題向け

## コンテキストウィンドウ管理

最後の 20% を避ける:
- 大規模リファクタ
- 複数ファイル機能実装
- 複雑相互作用のデバッグ

## Extended Thinking + Plan Mode

複雑タスクでは:
1. extended thinking を有効化（既定有効）
2. **Plan Mode** を有効化
3. 複数回の批評ラウンドを回す
4. 役割分担サブエージェントで多面的に検証

## ビルドトラブルシューティング

1. **build-error-resolver** を使う
2. エラーを分析する
3. 段階的に修正する
4. 各修正後に再検証する
