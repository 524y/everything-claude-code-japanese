# セキュリティガイドライン

## 必須セキュリティチェック

コミット前に:
- [ ] ハードコードされたシークレットがない
- [ ] 全ユーザー入力を検証している
- [ ] SQL インジェクション対策がある
- [ ] XSS 対策がある
- [ ] CSRF 保護がある
- [ ] 認証 / 認可を確認済み
- [ ] 全エンドポイントにレート制限
- [ ] エラーで機密情報を漏らさない

## シークレット管理

- ソースコードへ直接埋め込まない
- 環境変数または secret manager を使う
- 起動時に必須シークレットの存在を検証
- 露出したシークレットはローテーションする

## セキュリティ対応プロトコル

問題検知時:
1. 直ちに停止
2. **security-reviewer** を使う
3. CRITICAL を修正してから再開
4. 露出シークレットをローテーション
5. 類似問題を全体で再監査
