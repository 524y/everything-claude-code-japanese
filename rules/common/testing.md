# テスト要件

## 最小テストカバレッジ: 80%

必要なテスト種別:
1. **ユニットテスト**
2. **統合テスト**
3. **E2E テスト**（言語 / フレームワークに応じた適切な実装）

## テスト駆動開発

1. RED: 先にテストを書く
2. FAIL を確認
3. GREEN: 最小実装
4. PASS を確認
5. REFACTOR
6. 80%+ カバレッジを確認

## テスト失敗時

1. **tdd-guide** を使う
2. テスト分離を確認
3. モック設定を確認
4. テストが誤っていない限り実装側を修正

## エージェント支援

- **tdd-guide**: 新機能で積極利用し、write-tests-first を徹底する
