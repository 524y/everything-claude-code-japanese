# テスト要件

## 最小テストカバレッジ: 80%

テスト種別 (すべて必須):
1. **ユニットテスト** - 個別の関数、ユーティリティ、コンポーネント
2. **統合テスト** - API エンドポイント、データベース操作
3. **E2E テスト** - 重要なユーザーフロー (Playwright)

## テスト駆動開発

必須ワークフロー:
1. テストを先に書く (RED)
2. テストを実行する - 失敗するはず
3. 最小限の実装を書く (GREEN)
4. テストを実行する - 通るはず
5. リファクタする (IMPROVE)
6. カバレッジを確認する (80%+)

## テスト失敗のトラブルシューティング

1. **tdd-guide** エージェントを使う
2. テストの分離を確認する
3. モックが正しいか検証する
4. テストが誤っている場合を除き、実装を修正する

## エージェント支援

- **tdd-guide** - 新機能では PROACTIVELY に使い、write-tests-first を強制する
- **e2e-runner** - Playwright E2E テストの専門家
