# /learn - 再利用可能なパターンの抽出

現在のセッションを分析し、スキルとして保存する価値のあるパターンを抽出する。

## トリガー

非トリビアルな問題を解決したタイミングで `/learn` を実行する。

## 抽出する内容

次を探す:

1. **エラー解決パターン**
   - どんなエラーが発生したか?
   - 根本原因は何か?
   - 何で直ったか?
   - 同様のエラーに再利用できるか?

2. **デバッグ手法**
   - 非自明なデバッグ手順
   - 効果的だったツールの組み合わせ
   - 診断パターン

3. **回避策**
   - ライブラリの癖
   - API の制約
   - バージョン固有の修正

4. **プロジェクト固有パターン**
   - 発見したコードベースの規約
   - 決定したアーキテクチャ
   - 統合パターン

## 出力形式

`~/.claude/skills/learned/[pattern-name].md` にスキルファイルを作成する:

```markdown
# [パターン名]

**Extracted:** [日付]
**Context:** [適用される状況の説明]

## Problem
[解決する問題 - 具体的に書く]

## Solution
[パターン / 手法 / 回避策]

## Example
[必要ならコード例]

## When to Use
[このスキルを使うべきトリガー条件]
```

## プロセス

1. セッションから抽出可能なパターンをレビューする
2. 最も価値が高く再利用可能な洞察を特定する
3. スキルファイルの下書きを作成する
4. 保存前にユーザーへ確認する
5. `~/.claude/skills/learned/` に保存する

## 注意

- 些細な修正（typo、単純な構文エラー）は抽出しない
- 一度きりの問題（特定の API 障害など）は抽出しない
- 将来のセッションで時間を節約できるパターンに集中する
- スキルは 1 つのパターンに絞る
